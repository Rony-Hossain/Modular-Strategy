How Anthropic API billing actually works
You add a payment method, then:

Pre-purchase credits (recommended) — buy $5 / $20 / $100 of credit upfront, it depletes as you use it. When it hits zero, calls fail until you top up.
Auto-reload — set a threshold (e.g. "when balance drops below $5, charge me another $20")
Spending limits — hard cap. Calls get rejected once you hit it. Set this to $20 and you literally cannot overspend, regardless of bugs.
The closest thing to a "subscription"
Pre-purchase $20 in credits + set a $20 monthly spending limit. Functionally that's a $20/month ceiling. If you don't use it all, the credit rolls over. If you do use it all, calls stop until next month or until you add more.

This gives you:

Predictable maximum cost ($20/month worst case)
No commitment to spend $20 if you don't use it
Hard guarantee that bugs can't bankrupt you

API consumption varies by orders of magnitude between users — some make 10 calls/day, some make 10,000.
